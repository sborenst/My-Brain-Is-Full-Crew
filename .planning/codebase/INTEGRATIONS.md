# External Integrations

**Analysis Date:** 2026-03-24

## APIs & External Services

**Email & Calendar:**
- Gmail - Email reading, searching, message retrieval, draft creation
  - SDK/Client: MCP (Model Context Protocol) server at `https://gmail.mcp.claude.com/mcp`
  - Auth: Automatic via Claude Code (user authorizes when first needed)
  - Agent: `postman.md`
  - Functions: `gmail_search_messages`, `gmail_read_message`, `gmail_read_thread`, `gmail_create_draft`

- Google Calendar - Event reading, event creation, deadline tracking
  - SDK/Client: MCP server at `https://gcal.mcp.claude.com/mcp`
  - Auth: Automatic via Claude Code (user authorizes when first needed)
  - Agent: `postman.md`
  - Functions: `gcal_list_events`, calendar event creation

## Data Storage

**Vault Storage:**
- Local filesystem (Obsidian vault directory structure)
- Storage location: User-designated Obsidian vault folder
- Format: Markdown files with YAML frontmatter
- Structure: PARA methodology (Projects, Areas, Resources, Archive)

**Key directories:**
- `00-Inbox/` - Capture point for all new notes
- `01-Projects/` - Active projects and their progress
- `02-Areas/` - Life areas (Work, Health, Learning, etc.)
- `03-Resources/` - Reference materials and resources
- `04-Archive/` - Completed projects and old content
- `MOC/` - Master Index and area maps of content
- `Templates/` - Note templates for consistency
- `Meta/` - System configuration and user profile

**File Format:**
- Markdown (`.md`) with YAML frontmatter
- Frontmatter fields: `type`, `date`, `tags`, `status`, `priority`, `deadline`, etc. (context-specific)

## Email & Calendar Integration (Postman Agent)

The Postman agent (`agents/postman.md`) is the primary integration point. It operates in multiple modes:

### Email Operations

**Scan Gmail Inbox:**
- Query: `is:inbox is:unread` or time-limited queries with `after:{{yesterday}}`
- Process: Read full thread, priority scoring based on sender importance and content signals
- Save: Structured notes to `00-Inbox/` with types:
  - `email-action` - Requires user response
  - `email-deadline` - Contains deadline
  - `email-info` - Informational only
  - `email-financial` - Invoice or receipt
  - `email-meeting` - Meeting invitation

**Email Filtering (Exclusions):**
- Newsletters, mailing lists, marketing
- Automated notifications (unless critical failures)
- System emails (password resets, 2FA)
- Trivial receipts and confirmations

**VIP Email Processing:**
- Defined in `Meta/user-profile.md`
- Always saved regardless of content
- Searched with `from:{{vip-email}}` queries

**Email Draft Creation:**
- Mode: "Email Draft" or "Draft Reply"
- Process: Read context, synthesize response, create draft in Gmail
- User must review and send manually (no auto-send)

### Calendar Operations

**Calendar Import:**
- Retrieves events from Google Calendar
- Saves as structured notes to `00-Inbox/`
- Format: Time, location, attendees, description
- Tags with `#calendar` and event-specific tags

**Create Calendar Event:**
- Triggered by user request or note-based event
- Uses Google Calendar API to create events
- Automatically linked from vault notes

**Deadline Radar:**
- Scans all emails for deadline keywords: "deadline", "due by", "scadenza", "expires", etc.
- Scans Google Calendar for upcoming deadline-like events
- Generates deadline report in `00-Inbox/`

**Meeting Prep:**
- Gathers context from previous meetings, related emails, vault notes
- Creates structured meeting note with prepared talking points
- Saves to `00-Inbox/` and can link to relevant project/area

**Weekly Agenda:**
- Scans Calendar for week (Monday-Sunday)
- Extracts important events and deadlines
- Generates comprehensive weekly overview note

## Vault Integration Points

All agents write to Obsidian vault using standardized patterns:

**Inbox (00-Inbox/):**
- Initial capture point for: emails, transcriptions, brainstorm notes, calendar imports
- Sorted by agents after initial capture
- Filename pattern: `{Type} - {Brief Title} - {ISO Date}.md`

**Project & Area Links:**
- All notes can link to projects (`01-Projects/{name}/`) or areas (`02-Areas/{area}/{sub-area}/`)
- Bidirectional links maintained by Connector agent
- Cross-references validated by Librarian agent

**Template Application:**
- `Templates/{note-type}.md` - Used as scaffolding for structured notes
- Frontmatter copied by Scribe and other agents
- Maintains consistency across vault

**MOC (Map of Content):**
- `MOC/Master Index.md` - Links to all major areas
- `MOC/{Area}.md` - Links to all notes in that area
- Maintained by Architect and Connector agents

## Data Flow

```
User Input → Dispatcher (CLAUDE.md) → Appropriate Agent
                                            ↓
                                    Read/Write Vault
                                            ↓
                                    Email/Calendar API (if Postman)
                                            ↓
                                    Vault files updated
                                            ↓
                                    Inter-agent suggestions
                                            ↓
                                    Next agent chained (if needed)
```

## Limitations & Considerations

**Gmail/Calendar:**
- Read-only for email (except draft creation)
- No automatic email sending
- Rate limits on Gmail API (prioritizes VIP emails if exceeded)
- Requires active Google account with Gmail and Calendar enabled

**Vault:**
- Single-user system (personal vault only)
- Local-only (no cloud sync built-in; user can use Obsidian Sync or git)
- No multi-vault support
- GDPR compliance is user's responsibility if processing others' data

**API Compliance:**
- No responsibility for Gmail/Calendar API ToS violations
- User responsible for compliance with third-party data usage (e.g., GDPR for emails from others)
