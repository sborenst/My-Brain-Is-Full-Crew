# Technology Stack

**Analysis Date:** 2026-03-24

## Languages

**Primary:**
- Markdown - Agent system prompts and documentation (`.md` files)
- Bash - Installation and update scripts (`launchme.sh`, `updateme.sh`)
- Python - Code generation utility for skills (`scripts/generate-skills.py`)
- YAML - Configuration and frontmatter in agent/reference files

**Secondary:**
- JSON - Plugin manifest and MCP configuration

## Runtime

**Environment:**
- Claude Code (Anthropic's official CLI for Claude)
- Claude Models: `opus`, `sonnet` (assigned per-agent)
- Obsidian vault (local file-based knowledge management system)

**Package Manager:**
- Bash shell (system-level)
- Not applicable for Python dependencies (single script utility)

## Frameworks

**Core:**
- Claude Code Framework - AI subagent system with dispatcher orchestration
- Obsidian - Local markdown vault system with YAML frontmatter

**Model Assignment:**
- Architect: `opus` - handles complex structural decisions
- Librarian: `opus` - handles vault audits and complex analysis
- Scribe: `sonnet` - note capture and formatting
- Sorter: `sonnet` - inbox triage and filing
- Seeker: `sonnet` - vault search and intelligence
- Connector: `sonnet` - link analysis and graph operations
- Transcriber: `sonnet` - audio/transcription processing
- Postman: `sonnet` - email and calendar integration

**Tooling:**
- Claude Code Plugin System - Manifest-based agent registration at `.claude-plugin/plugin.json`
- Subagent Framework - Auto-loaded agents from `.claude/agents/` directory
- Skills Generation - Python script generates compatible format for Claude Code Desktop

## Key Dependencies

**Critical:**
- Claude Code CLI - Base runtime for agent orchestration
- Obsidian - Local vault storage (user provides)
- Bash shell - For installation and agent setup scripts

**Integration Providers (Optional):**
- Gmail API (via MCP) - Email reading, searching, drafting, creating events
- Google Calendar API (via MCP) - Calendar event reading and creation

**Utilities:**
- Python 3 - For skill generation (optional; skipped gracefully if unavailable)

## Configuration

**Environment:**
- No environment variables required for core operation
- Optional Gmail/Google Calendar configuration during setup
- MCP configuration in `.mcp.json` (created by installer if integrations enabled)

**Build/Generation:**
- Agent system prompts are authored directly in `agents/*.md`
- Skills auto-generated by `scripts/generate-skills.py` for Claude Code Desktop compatibility
- Installation via `bash scripts/launchme.sh` copies agents and references into vault's `.claude/` directory
- Update via `bash scripts/updateme.sh` syncs agent files

**Configuration Files:**
- `.claude-plugin/plugin.json` - Plugin manifest for `claude --plugin-dir` usage
- `.mcp.json` - MCP server configuration (created during setup if Gmail/Calendar selected)
- `agents/*.md` - Frontmatter contains tool access and model selection per agent

## Platform Requirements

**Development:**
- Obsidian (any version supporting markdown vaults)
- Claude Code (requires Claude Pro, Max, or Team subscription)
- Bash shell (Linux, macOS, or WSL on Windows)
- Python 3 (optional, for skill generation on Desktop/Cowork)

**Production (User-facing):**
- Obsidian vault (local directory on user's machine)
- Claude Code CLI or Claude Code Desktop (Cowork)

**Deployment Target:**

## Installation Flow

```bash
# Step 1: Clone repo into Obsidian vault
cd /path/to/your-vault
git clone https://github.com/gnekt/My-Brain-Is-Full-Crew.git

# Step 2: Run installer
cd My-Brain-Is-Full-Crew
bash scripts/launchme.sh

# Installer:
# - Creates .claude/agents/ with 8 agent files
# - Creates .claude/references/ with shared documentation
# - Generates .claude/skills/ (for Desktop/Cowork, if python3 available)
# - Optionally creates .mcp.json for Gmail/Calendar integration
```

## Runtime Configuration

- No npm, pip, or dependency managers needed
- Agent frontmatter specifies exact tools and model per subagent
- Dispatcher (`CLAUDE.md`) routes user messages to appropriate agent based on priority rules
- Inter-agent coordination via suggested-next-agent output pattern
